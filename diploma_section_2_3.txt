2.3. Разработка клиентской части

Клиентская часть представляет собой интерфейс, с которым непосредственно взаимодействует
покупатель в браузере. В проекте применяется смешанный подход: React Server Components
отвечают за загрузку данных на сервере и формирование первоначального HTML-ответа, тогда
как Client Components (помеченные директивой 'use client') обеспечивают интерактивность —
открытие меню, управление корзиной, переключение темы оформления.

Основные маршруты приложения:

— app/page.tsx — главная страница. Серверный компонент выполняет параллельные запросы
  к базе данных через Prisma: загружаются категории, баннеры, подборки товаров
  (рекомендуемые, новинки, хиты продаж) и опубликованные отзывы. Результат передаётся
  дочерним компонентам в виде пропсов.

— app/catalog/page.tsx и app/catalog/[...slug]/page.tsx — страницы каталога и вложенных
  категорий с фильтрацией и пагинацией.

— app/product/[slug]/page.tsx — карточка конкретного товара с характеристиками,
  изображениями, вариантами и отзывами.

— components/ — библиотека переиспользуемых компонентов: Header, Footer, ProductCard,
  ProductGrid, HeroBanner, CategoryGrid, TradeInBlock, ImeiCheckBlock и другие.

Задачи клиентской части:

— наглядное отображение каталога и фильтров;
— предсказуемая работа корзины (состояние на клиенте с синхронизацией с сервером через
  REST API);
— адаптивная вёрстка для мобильных устройств и десктопа;
— единый визуальный стиль на основе Tailwind CSS и общих компонентов.

Шапка сайта (components/Header.tsx) реализует навигацию по категориям (iPhone, iPad,
MacBook, AirPods, Apple Watch, Аксессуары), строку поиска, иконки избранного, сравнения
и корзины, работу с сессией пользователя через NextAuth (хук useSession), а также
переключатель темы оформления.

серверные компоненты обеспечивают быструю первоначальную загрузку и SEO, клиентские —
отзывчивый пользовательский интерфейс без лишних перезагрузок страницы.
