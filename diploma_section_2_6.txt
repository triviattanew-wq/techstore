2.6. Размещение веб-сайта на хостинге

Хостинг для Next.js-приложения должен поддерживать среду выполнения Node.js
и возможность запуска команды сборки (npm run build) и сервера (npm run start)
либо использовать платформу с нативной поддержкой Next.js (Vercel, Netlify,
Railway, Render и аналоги).

Типовой порядок развёртывания на VPS или выделенном сервере:

1. Создание базы данных PostgreSQL и получение строки подключения DATABASE_URL.
   Для локальной разработки используется Docker Compose с контейнером postgres:15-alpine.
   Для продакшена рекомендуется управляемая база данных (AWS RDS, DigitalOcean
   Managed Databases, Supabase и т.п.).

2. Настройка переменных окружения в файле .env или через панель хостинга:
   — DATABASE_URL — строка подключения к PostgreSQL
   — NEXTAUTH_SECRET — секретный ключ для NextAuth (генерируется через openssl)
   — NEXTAUTH_URL — публичный URL приложения (например, https://techstore.ru)
   — ADMIN_EMAIL, ADMIN_PASSWORD — учётные данные администратора
   — NEXT_PUBLIC_SITE_URL, NEXT_PUBLIC_SITE_NAME — публичные переменные для клиента
   — NODE_ENV=production — режим продакшена

3. Выполнение миграций Prisma для создания структуры базы данных:
   npx prisma migrate deploy
   Либо для быстрого прототипирования:
   npx prisma db push

4. Заполнение начальных данных (категории, бренды, товары, администратор):
   npm run db:seed
   Скрипт выполняет файл prisma/seed.js, который создаёт тестовые данные
   и учётную запись администратора.

5. Сборка приложения для продакшена:
   npm run build
   Next.js генерирует оптимизированные статические и серверные страницы
   в директорию .next/

6. Запуск процесса Node.js:
   npm run start
   Приложение запускается на порту 3000 (по умолчанию). Для продакшена
   рекомендуется использовать менеджер процессов PM2:
   pm2 start npm --name "techstore" -- start
   pm2 save
   pm2 startup

7. Настройка обратного прокси-сервера nginx с TLS-сертификатом (Let's Encrypt):
   — nginx проксирует запросы с порта 80/443 на localhost:3000
   — настраивается автоматическое перенаправление HTTP → HTTPS
   — устанавливается SSL-сертификат через certbot
   — включается сжатие gzip и кэширование статики

8. Финальная проверка в браузере:
   — публичные маршруты: главная, каталог, карточка товара, корзина
   — административные маршруты: /admin/login, /admin/products, /admin/leads
   — проверка работы форм: оформление заказа, Trade-in, IMEI-check
   — тестирование на мобильных устройствах

Альтернативный вариант развёртывания — использование платформы Vercel
(создатели Next.js). В этом случае шаги 5-7 выполняются автоматически:
достаточно подключить GitHub-репозиторий, указать переменные окружения
и нажать Deploy. Vercel автоматически выполняет сборку, настраивает CDN,
выдаёт SSL-сертификат и предоставляет домен вида project-name.vercel.app.

Для данного дипломного проекта использовался локальный тестовый контур
с Docker Compose для PostgreSQL и запуском Next.js в режиме разработки
(npm run dev) на порту 3000.

Таким образом, процесс развёртывания Next.js-приложения включает настройку
базы данных, переменных окружения, выполнение миграций, сборку и запуск
сервера с последующей настройкой веб-сервера и SSL-сертификата для
обеспечения безопасного доступа.
