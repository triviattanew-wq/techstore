ВВЕДЕНИЕ
(параграфы 1–3 из вашего варианта можно оставить с минимальными правками)

Современная торговля электроникой всё активнее переходит в цифровую среду. Покупатели ожидают, что интернет-магазин позволит быстро найти нужную модель устройства, сравнить характеристики, уточнить наличие, выбрать способ доставки и оформить заказ без обращения к консультанту. Особенно актуально это для техники Apple, поскольку товары данной категории отличаются высокой стоимостью, большим количеством модификаций, зависимостью цены от объёма памяти, цвета, поколения устройства и комплектации.

Интернет-магазин техники Apple должен решать не только задачу публикации карточек товаров, но и задачу комплексной автоматизации торгового процесса. Система должна обеспечивать каталогизацию продукции, удобный поиск, фильтрацию по характеристикам, управление корзиной, оформление заявки на покупку (заказа), хранение истории обращений, обработку заявок администратором и учёт складских остатков.

Актуальность темы дипломного проекта обусловлена необходимостью разработки удобного и функционального веб-приложения, ориентированного на продажу смартфонов, ноутбуков, планшетов, часов, наушников и аксессуаров Apple.

Практическая значимость проекта заключается в создании программного продукта на базе современного стека веб-разработки (fullstack-приложение на JavaScript/TypeScript), который может быть использован небольшим или средним магазином электроники для организации онлайн-продаж.

Проблема исследования: отсутствие у магазина техники Apple единого интернет-магазина, обеспечивающего структурированный каталог товаров, онлайн-оформление заявок, управление корзиной, учёт обращений и администрирование ассортимента.

Цель работы: разработка интернет-магазина техники Apple, обеспечивающего просмотр каталога, выбор товаров, оформление заявок и администрирование торговых процессов.

Задачи:
• изучить особенности организации интернет-торговли техникой Apple;
• провести анализ аналогичных интернет-магазинов;
• определить функциональные и нефункциональные требования к разрабатываемому веб-сайту;
• выполнить анализ технического задания и выбрать технологии реализации;
• разработать дизайн интерфейса в графическом редакторе;
• реализовать клиентскую часть веб-приложения;
• реализовать серверную часть, включая API, базу данных и административные функции;
• спроектировать структуру каталога и связанных сущностей в базе данных;
• разместить веб-сайт на хостинге (или подготовить сборку к развёртыванию);
• выполнить мероприятия по SEO-оптимизации;
• провести тестирование и оценить эффективность внедрения.

Объект исследования: деятельность интернет-магазина, специализирующегося на продаже техники Apple.

Предмет исследования: процесс разработки веб-приложения для автоматизации продажи техники Apple.

1.4. Описание обеспечивающей части веб-сайта (ЗАМЕНИТЬ раздел про WordPress на следующий текст)

В процессе проектирования интерфейса используется графический редактор Figma. Данный инструмент позволяет создавать макеты страниц, интерактивные прототипы и пользовательские сценарии до начала программной реализации. Прототипирование помогает проверить удобство структуры, расположение элементов управления, логику переходов и последовательность оформления заказа.

Для подготовки графических элементов применяются векторные и растровые редакторы. Иконки категорий, баннеры и элементы интерфейса выполняются в едином визуальном стиле, соответствующем минималистичной эстетике брендов премиальной электроники.

В качестве основы программной реализации выбрано веб-приложение на платформе Next.js (версия 14) с использованием маршрутизации App Router. Next.js представляет собой React-фреймворк с серверными компонентами и маршрутизацией на основе файловой структуры каталога app/, что упрощает организацию страниц каталога, карточки товара, корзины и административного раздела. Документация Next.js описывает режимы рендеринга, кэширования и работу с серверными действиями, что важно для производительного интернет-магазина.

Клиентский интерфейс реализуется с помощью библиотеки React 18 и языка TypeScript, обеспечивающего типобезопасность и снижение количества ошибок при разработке. Стилизация выполняется утилитарным CSS-фреймворком Tailwind CSS, позволяющим быстро собирать адаптивную вёрстку и поддерживать единообразие компонентов (кнопки, карточки товаров, сетки каталога).

Серверная логика и обмен данными с клиентом реализуются через API Route Handlers в каталоге app/api/ — это стандартный механизм Next.js для создания REST-подобных конечных точек на одном сервере с фронтендом. Для доступа к базе данных используется Prisma ORM и СУБД PostgreSQL. Prisma обеспечивает декларативное описание моделей данных, миграции схемы и типизированный клиент запросов к базе.

Аутентификация пользователей реализуется с помощью NextAuth.js (интеграция с Prisma Adapter), что позволяет использовать безопасные сессии и разграничение прав для личного кабинета и административной части.

Размещение готового приложения может осуществляться на платформе, поддерживающей Node.js (например, облачный хостинг с Docker или специализированные PaaS-сервисы для Next.js), с отдельным экземпляром PostgreSQL и настройкой переменных окружения (строка подключения к базе, секреты сессии). Обязательным условием для обработки персональных данных остаётся использование протокола HTTPS.

Таким образом, обеспечивающая часть включает: средства проектирования UI (Figma), средства разработки (Visual Studio Code или аналог), стек Next.js + React + TypeScript + Tailwind CSS, Prisma и PostgreSQL, NextAuth.js, а также инструменты сборки и развёртывания npm-скриптов проекта (dev/build/start).

ГЛАВА 2. РАЗРАБОТКА ИНТЕРНЕТ-МАГАЗИНА ТЕХНИКИ APPLE

2.1. Анализ технического задания

Анализ технического задания выполняется с целью сопоставить требования заказчика (магазина) с реальными решениями в коде и инфраструктуре. По результатам анализа сформулированы следующие требования:

• веб-сайт должен содержать информацию о магазине, каталог товаров с фильтрами и поиском, карточку товара с вариантами (цвет, память и т.д.), корзину и форму оформления заявки;
• функциональные блоки: главная страница, каталог, страница товара, корзина, оформление заявки, личный кабинет (для зарегистрированного пользователя), административная панель;
• шапка сайта должна содержать логотип/название, навигацию по категориям, поиск, иконки «избранное», «сравнение», «корзина», вход в учётную запись;
• подвал — контакты, ссылки на информационные страницы, юридическая информация;
• дизайн — современный адаптивный интерфейс, корректное отображение на ПК, планшете и смартфоне;
• безопасность: разграничение ролей, защита административных маршрутов, хранение паролей в виде хеша, использование HTTPS;
• производительность: разумное количество запросов к БД, индексы в схеме данных для частых выборок по категории, бренду, цене, признакам «новинка/хит».

Для реализации выбраны следующие технологии:

• проектирование макетов — Figma;
• клиентская часть — HTML (разметка JSX), CSS (в т.ч. через Tailwind CSS), язык TypeScript, библиотека React 18, фреймворк Next.js 14 (App Router);
• серверная часть — маршруты API в Next.js, Prisma ORM, СУБД PostgreSQL;
• аутентификация — NextAuth.js с адаптером для Prisma;
• дополнительные библиотеки: валидация форм (react-hook-form, zod), интерфейсные примитивы (Radix UI), анимации (Framer Motion), иконки (lucide-react), SEO (next-seo) — по мере необходимости модулей проекта.

Обоснование отказа от CMS WordPress: для учебного и коммерческого проекта интернет-магазина требуется гибкая бизнес-логика фильтрации товаров, вариантов, складских остатков и заявок; связка Next.js + Prisma даёт полный контроль над схемой данных и API, а также единый язык TypeScript на клиенте и сервере.


2.2. Разработка дизайна

Дизайн-концепция отражает минималистичный стиль, крупные изображения устройств, читаемую типографику и акцентные кнопки призыва к действию.

На первом этапе в Figma разрабатываются сетки под ключевые разрешения (desktop / tablet / mobile), структура главной страницы (баннер, категории, подборки товаров), страницы каталога с боковой или выдвижной панелью фильтров, карточка товара, корзина, форма заявки, админ-экраны списков товаров и заявок.

На втором этапе дорабатываются компоненты: состояния кнопок, пустая корзина, загрузка, ошибки форм. Для согласования с заказчиком используются интерактивные прототипы переходов в Figma.

Рисунки: разместите скриншоты из Figma (как у Николаевой: «Разработка макета в Figma», «Дизайн для ПК/планшета/смартфона»).


2.3. Разработка клиентской части

Клиентская часть — это интерфейс, с которым взаимодействует покупатель в браузере. В проекте используется смешение React Server Components (загрузка данных на сервере для первого отображения страницы) и Client Components (интерактивность: меню, корзина, переключение темы), помеченных директивой 'use client'.

Основные маршруты приложения (примеры из структуры проекта tech-store):
• app/page.tsx — главная страница: запросы к базе через Prisma для категорий, баннеров, подборок товаров (рекомендуемые, новинки, хиты), отзывов;
• app/catalog/page.tsx и app/catalog/[...slug]/page.tsx — каталог и вложенные категории;
• app/product/[slug]/page.tsx — карточка товара;
• компоненты в каталоге components/ — Header, Footer, сетки товаров, баннеры, блоки Trade-in и проверки IMEI и др.

Задачи клиентской части:
• наглядное отображение каталога и фильтров;
• предсказуемая работа корзины (контекст состояния на клиенте + синхронизация с сервером при необходимости);
• адаптивная вёрстка;
• единый визуальный стиль (Tailwind + общие компоненты).

Пример: шапка сайта Header.tsx реализует навигацию по фиксированному списку категорий (iPhone, iPad, MacBook, AirPods, Apple Watch, Аксессуары), поиск, иконки избранного, сравнения и корзины, работу с сессией NextAuth (useSession), переключение темы (ThemeToggle).

Рисунки: вставьте фрагмент JSX/TSX из компонента карточки товара или секции главной страницы; скриншот главной в браузере.


2.4. Разработка серверной части

Серверная часть обрабатывает HTTP-запросы, выполняет бизнес-логику, обращается к базе данных и формирует ответы JSON для клиента либо готовый HTML при SSR.

Ключевые элементы:
• lib/prisma.ts — единый экземпляр Prisma Client;
• app/api/products/route.ts — выборка товаров с фильтрами (категория, бренды, диапазон цены, память, цвет, признаки в наличии/новинка/хит, сортировка, пагинация, текстовый поиск);
• app/api/cart/route.ts — операции корзины;
• app/api/admin/* — административные операции (товары, категории, бренды, заявки, статистика) с проверкой прав;
• middleware.ts — при необходимости задание заголовков кэширования и ограничение matcher для публичных и админ-маршрутов.

Аутентификация и роли: в схеме Prisma предусмотрены роли пользователя UserRole (USER, ADMIN, MANAGER, CONTENT_MANAGER) и отдельная модель AdminUser для входа в админ-панель; NextAuth обеспечивает сессии и безопасное хранение учётных данных согласно настройкам адаптера.

Безопасность: пароли хранятся как passwordHash (алгоритм хеширования задаётся при регистрации — bcrypt и аналоги), административные API защищены проверкой сессии/роли, формы проходят валидацию (zod) на сервере.

Рисунки: скриншот админ-списка товаров или заявок; схема маршрутов API (можно нарисовать в draw.io).


2.5. Проектирование структуры каталога и данных о товарах

Логическая модель данных описана в файле prisma/schema.prisma и включает сущности, отражающие предметную область магазина Apple:

• User — пользователь сайта, роль, связь с корзиной, избранным, отзывами, заявками;
• Category — категория с иерархией parentId, SEO-полями, сортировкой;
• Brand — бренд (для Apple и сопутствующих производителей аксессуаров);
• Product — товар: цена, старая цена, описание, связь с категорией и брендом, признаки isFeatured/isNew/isHit, SEO, гарантия, информация о доставке;
• ProductImage, ProductVariant (цвет, память, тип SIM, цена варианта, остаток), ProductCharacteristic (имя/значение характеристики);
• Stock, Warehouse — учёт количества по складам;
• Cart, CartItem — корзина (по пользователю или sessionId);
• Lead, LeadItem — заявка/заказ, состав, статус (NEW, IN_PROGRESS, CONFIRMED, CANCELLED, COMPLETED);
• Review, ReviewImage — отзывы и модерация isPublished;
• Favorite, CompareItem — избранное и сравнение;
• Banner, PromoCode, BlogPost, Page — контент и маркетинг;
• TradeInRequest, ImeiCheckRequest, ContactRequest — дополнительные сервисы магазина.

Связи «категория — товары», «товар — варианты», «заявка — позиции» реализуют классические отношения один-ко-многим и многие-ко-многим через промежуточные таблицы.

Рисунок: ER-диаграмма (Prisma Studio или экспорт из инструмента моделирования).


2.6. Размещение веб-сайта на хостинге

Хостинг для Next.js-приложения должен поддерживать среду Node.js и возможность запуска команды сборки (npm run build) и сервера (npm run start) либо использовать платформу с нативной поддержкой Next.js.

Типовой порядок развёртывания:
1) создание базы данных PostgreSQL и получение строки подключения DATABASE_URL;
2) настройка переменных окружения (DATABASE_URL, NEXTAUTH_SECRET, NEXTAUTH_URL, параметры почты при использовании e-mail);
3) выполнение миграций Prisma (prisma migrate deploy) или prisma db push на тестовом контуре;
4) заполнение начальных данных (prisma db seed);
5) сборка приложения (npm run build);
6) запуск процесса Node и настройка обратного прокси (nginx) с TLS-сертификатом;
7) финальная проверка в браузере публичных и админ-маршрутов.

Рисунок: скриншот работающего сайта на домене (если есть) или локально с указанием, что для диплома использовался тестовый контур.


2.7. SEO-оптимизация

Для интернет-магазина важны: уникальные заголовки и описания страниц, человеко-понятные URL (slug товара и категории), семантическая разметка, файл robots.txt, карта сайта sitemap.xml, корректные статусы ответов сервера.

В проекте подключена библиотека next-seo; метаданные страниц задаются через generateMetadata / мета-теги Next.js 14. Рекомендуется зарегистрировать сайт в Яндекс.Вебмастер и Google Search Console, выгрузить sitemap.

Рисунки: скриншот настроек мета-тегов; пример robots.txt и sitemap (если сгенерированы).


2.8. Тестирование веб-сайта

В ходе выполнения дипломного проекта разработано веб-приложение интернет-магазина техники Apple на стеке Next.js, React, TypeScript, Tailwind CSS, Prisma и PostgreSQL с использованием NextAuth.js для аутентификации.
